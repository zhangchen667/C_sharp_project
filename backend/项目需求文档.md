# **软件工程大作业 - "我的个人空间" 需求文档**

## **1. 项目内容**

本项目为基于 **C# ASP.NET Core Web API (.NET 9.0)** + **Vue 3** 前后端分离架构开发的个人综合空间系统。系统采用面向对象（OOP）设计思想，后端提供 RESTful API 接口，前端使用 Vue 3 单页应用（SPA）进行交互，实现个人博客、多媒体相册、访客留言及后台管理功能。系统使用 **ASP.NET Core Identity** + **JWT** 实现用户认证与授权，后端使用 **EF Core (Pomelo)** 连接 **MySQL 8.0** 数据库。项目旨在通过 4 周的敏捷开发，实践软件工程全生命周期管理，并深度集成数据库、Linq 查询、异步编程等核心技术。

## **2. 技术栈**

| 层次 | 技术选型 | 说明 |
| :--- | :--- | :--- |
| 开发语言 | C#（后端）/ JavaScript（前端） | 后端核心业务逻辑，前端页面交互 |
| Web 框架（后端） | ASP.NET Core Web API (.NET 9.0) | RESTful API 接口，返回 JSON 数据 |
| 前端框架 | Vue 3 + Vite | 单页应用（SPA），前后端分离架构 |
| UI 框架 | Element Plus | Vue 3 组件库，提供丰富的 UI 组件 |
| 状态管理 | Pinia | Vue 3 官方推荐的状态管理库 |
| 路由 | Vue Router | 前端单页应用路由管理 |
| HTTP 请求 | Axios | 前端与后端 API 通信 |
| 数据库 | MySQL 8.0 | 存储用户、文章、留言等数据 |
| ORM | EF Core + Pomelo.EntityFrameworkCore.MySql | 对象关系映射，支持 Linq 查询 |
| 身份认证 | ASP.NET Core Identity + JWT | 用户注册、登录、角色管理，JWT Token 无状态认证 |
| 跨域 | CORS | 前后端分离架构下的跨域请求支持 |

## **3. 系统整体功能**

* **用户与安全模块：**
  * 利用**正则表达式**实现注册信息的严格校验（如邮箱格式、强密码逻辑）。
  * 站长登录验证，确保后台管理权限安全。
* **个人博文模块：**
  * 支持文章的发布、分类及标签管理。
  * 集成 **Markdig** 解析器，支持 Markdown 格式编写博文，实时渲染为排版精美的技术长文。
  * 集成 **MathJax** 引擎，支持 LaTeX 数学公式的在线渲染。
  * 集成 **Linq** 技术实现文章的关键词检索、分类过滤及时间倒序排列。
* **多媒体相册模块：**
  * 利用**文件操作**技术实现照片的物理上传与存储。
  * 前台瀑布流展示，支持图片的点击放大预览与删除。
* **互动留言模块：**
  * 访客留言功能，后端通过**异步编程**处理高并发写入，保证页面不卡顿。
* **后台管理与监控：**
  * 全系统集成全局**异常处理**，记录操作日志，确保系统在高负载或非法输入下的稳定性。

## **4. 技术实现（对应考核指标）**

为满足大作业选型要求，本项目重点实现以下技术点：

| 考核项类型 | 对应技术点 | 在本项目中的具体应用 |
| :--- | :--- | :--- |
| **必选项** | **面向对象** | 定义 User、Post、Photo、Comment 等实体类，封装业务逻辑，利用继承和多态处理不同类型的回复。 |
| **必选项** | **数据库** | 使用 MySQL 8.0 存储用户数据与文章内容，通过 EF Core (Pomelo) 进行对象关系映射。 |
| **必选项** | **WebAPI** | 采用 ASP.NET Core Web API 编写 RESTful 风格接口，前端 Vue 3 通过 Axios 调用 API 进行数据交互。 |
| **必选项** | **异常处理** | 在 Controller/PageModel 层设置 try-catch 块，并配置全局异常中间件，捕获数据库连接错误或 IO 异常。 |
| **可选项 1** | **Linq** | 用于数据库查询优化，如：`_db.Articles.Where(a => a.IsPublic).OrderByDescending(a => a.Date)`，实现文章搜索与分类筛选。 |
| **可选项 2** | **文件操作** | 使用 IFormFile 接口接收上传的照片，并通过 FileStream 将其持久化到服务器硬盘。 |
| **可选项 3** | **正则表达式** | 编写 Regex 表达式校验注册邮箱、密码强度，并在 Markdown 渲染前过滤非法脚本标签。 |
| **可选项 4** | **异步编程** | 关键数据库读写操作采用 async/await 关键字，提升 Web 服务器并发处理能力。 |
| **特色功能** | **Markdown + 公式渲染** | 使用 Markdig 解析 Markdown 语法，MathJax 渲染 LaTeX 公式，打造专业级技术博客写作体验。 |

## **5. 主要业务流程**

1. **发布文章流程：** 站长通过 Vue 前端提交 JSON 数据 -> Axios 发送 POST 请求 -> **Web API** 接收请求 -> **JWT** 验证身份 -> **正则表达式**校验内容安全性 -> **Markdig** 将 Markdown 转为 HTML -> **异步**写入**数据库** -> **Linq** 更新首页文章列表 -> 前端 **MathJax** 渲染页面中的公式。
2. **相册管理流程：** 站长通过 Vue 前端上传图片 -> Axios 发送 FormData -> 后端通过**文件操作**（IFormFile）保存至服务器路径 -> 数据库记录路径 -> **异常处理**机制确保上传失败时文件不残留 -> 前端展示。
3. **访客留言流程：** 访客提交表单 -> Axios 发送 POST 请求 -> 后台捕获并**异步**保存 -> 站长后台通过 **Linq** 筛选出未读留言 -> **正则表达式**过滤恶意内容。

## **6. 人员分工（4人团队：2前端 + 2后端）**

### **后端开发组**

**后端 1张致远：数据库与数据架构层 (DBA & Architect)**
* 负责 **MySQL** 数据库的搭建、E-R 图设计和建表。
* 负责配置 C# 与 MySQL 的连接（EF Core + Pomelo），构建基础的数据访问层。
* 封装数据实体类（User、Post、Photo 等），为整个项目打好地基。
* **认领考核项：** **数据库**、**面向对象**（类的封装与继承）。

**后端 2赵志轩：业务逻辑与核心技术层 (Business Logic)**
* 编写各模块的 API Controller 代码（如点击"保存文章"后的处理逻辑），提供 RESTful 接口。
* 负责处理图片/文件的上传逻辑，实现多媒体作品集的本地保存。
* 处理后端的高并发读写请求，并在所有关键节点加上安全兜底。
* **认领考核项：** **WebAPI**（接口设计）、**Linq 查询**（文章分类、搜索检索）、**文件操作**（读写图片文件）、**异步编程**（数据库查询和文件保存）、**异常处理**（全局 try-catch 防崩溃）。

### **前端开发组**

**前端 1何天赐：UI 架构与页面开发 (UI/UX & Frontend)**
* 负责整个网站的"装修"。使用 Vue 3 + Element Plus 设计并编写网站的布局组件，统一下拉菜单、侧边栏、页脚的风格。
* 负责所有页面的排版与样式美化（各类按钮、输入框、列表展示等）。
* 确保网站在电脑和手机上都能响应式完美显示。
* **认领考核项：** 前端页面实现与后端 API 联调。

**前端 2张宸豪：特色功能交互与安全校验 (Features & Validation)**
* **核心技术攻坚：** 负责在 Vue 3 前端集成 **Markdown 解析器** 和 **MathJax (LaTeX 公式渲染)** 引擎，确保用户输入的文本能变成排版精美的技术长文。
* 负责用户交互体验优化，比如相册的瀑布流展示效果、图片的点击放大预览。
* 在前端页面（如注册、留言表单）实施严格的输入合法性校验，防止恶意代码注入。
* **认领考核项：** **正则表达式**（验证邮箱、密码强度，并在 Markdown 渲染前过滤非法脚本）。

## **7. 时间进度安排（4周双线协作）**

### **第 1 周（并行起步）**
* **后端组：** 后端 1 建好数据库、完成 E-R 图；后端 2 搭建 ASP.NET Core Web API 项目框架，编写基础增删改查 API 接口。
* **前端组：** 前端 1 搭建 Vue 3 + Vite + Element Plus 项目框架，设计带导航栏的布局组件；前端 2 跑通 Markdown 和公式渲染的 Demo 测试页。

### **第 2 周（功能对接 - 博客与留言）**
* **前后端会师：** 前端把写好的文章列表页、发布页的 UI 交给后端联调。后端 2 把 Linq 查出来的数据通过 RESTful API 传递给前端，前端通过 Axios 调用接口获取数据。
* 前后端一起搞定登录注册（JWT 认证）、发布文章、留言的核心流程。

### **第 3 周（难点攻坚 - 作品集与相册）**
* **后端组：** 后端 2 重点攻克照片的**文件操作**和上传 API 接口。
* **前端组：** 前端 1 使用 Element Plus 组件设计画廊页面样式；前端 2 编写相册的前端展示特效，并用**正则表达式**严查上传的文件名和留言板输入。

### **第 4 周（系统合体与交付）**
* **全员：** 后端排查**异常处理**漏洞；前端精修样式错位。
* **汇总：** 将代码合并，部署测试，截图撰写最终的答辩文档和 PPT。
