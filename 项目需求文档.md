

# **“我的个人空间” 需求文档**

## **1\. 项目内容**

本项目旨在利用 **C\# (ASP.NET Core)** 开发一个名为“我的个人空间”的Web应用程序。该平台是一个高度个性化的个人信息展示与日常记录系统，集成了博客记录、相册展示、个人简历和访客留言功能。

本项目不仅为用户提供一个云端的个人展示平台，同时也作为团队成员在软件工程实践中掌握 **Web全栈开发、数据库设计及敏捷开发流程** 的核心载体。

## **2\. 系统整体功能**

系统分为“前台展示”与“后台管理”两大部分，核心功能模块如下：

* **用户认证模块：** 实现站长（管理员）的登录与身份校验，确保后台管理功能的安全性。  
* **博客/文章模块：** 支持文章的发布、分类管理、标签设置及富文本内容展示。  
* **多媒体相册模块：** 支持图片的上传、重命名、分类存储及瀑布流式的前台展示。  
* **个人简历/主页：** 动态展示站长的基本信息、技能矩阵及项目经历。  
* **交互留言模块：** 允许访客在无需登录的情况下提交留言，站长可在后台进行审核与回复。  
* **后台管理看板：** 汇总统计文章数、留言数及网站访问基础数据。

## **3\. 主要业务流程**

针对Web项目的特性，核心业务流程设计如下：

* **流程一：内容发布（站长端）**  
  登录后台 \-\> 进入“文章编写”界面 \-\> 输入内容并上传附件 \-\> 系统后端验证并持久化至数据库 \-\> 前台页面实时更新。  
* **流程二：访客互动（访客端）**  
  进入留言板 \-\> 填写昵称与留言内容 \-\> 提交后存入数据库（默认为“待审核”状态） \-\> 站长审核通过后在页面显示。  
* **流程三：资源管理（相册）**  
  选择本地图片上传 \-\> 后端进行格式与大小校验 \-\> 存储文件流至服务器并记录路径 \-\> 相册列表动态刷新。

## **4\. 人员分工**

团队由4名成员组成，分工明确以确保高效协作：

* **成员 A（项目经理 & 后端开发）：** 负责数据库表结构设计（Entity Framework Core）、项目架构搭建及核心业务逻辑实现。  
* **成员 B（前端开发 & UI设计）：** 负责响应式布局设计（支持手机/PC访问）、前端页面开发及交互效果实现。  
* **成员 C（后端开发 & API设计）：** 负责 RESTful API 接口开发、文件上传系统逻辑及后端的安全性校验（防注入、权限控制）。  
* **成员 D（测试工程师 & 文档管理员）：** 负责编写单元测试与功能测试用例、汇总整理课程要求的各阶段文档（如测试报告）及最后的项目演示PPT。

## **5\. 时间进度安排（4周敏捷开发版）**

由于开发周期仅为4周，团队采取**高强度并行开发**模式：

| 周次 | 阶段目标 | 关键任务 |
| :---- | :---- | :---- |
| **第1周** | **需求与基建** | 完成需求定稿、数据库E-R图设计；搭建 ASP.NET Core 开发环境并初始化 Git 仓库。 |

| **第4周** | **交付准备** | 项目部署上线；整理最终技术文档；完成项目答辩 PPT 制作并提交全部代码成果。 |
